README

一、运行环境
主机（客户端运行环境）
 操作系统：Windows 11
 Python版本：Python 3.13.14
 依赖说明：仅使用Python标准库（socket、sys、time、struct、random、math、datetime），无需额外安装第三方包

虚拟机（服务器运行环境）
 操作系统：Windows 10
 Python版本：Python 3.14
 依赖说明：仅使用Python标准库，无需额外安装第三方包

 二、文件说明
| 文件名 | 作用 | 运行位置 |
|--------|------|----------|
| udpclient.py | 客户端程序，负责建立连接、按GBN协议发送数据、处理超时重传、输出统计信息 | 主机系统 |
| udpserver.py | 服务器程序，负责验证客户端身份、接收数据、模拟丢包、返回累积确认 | 虚拟机系统 |
| run_log.txt | 程序自动生成的运行日志，记录所有发包、收包、超时、重传事件，带毫秒级时间戳 | 两端各自生成 |

 三、配置说明
所有可配置参数均在代码开头集中定义，可根据需求修改：

 客户端（udpclient.py）
| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| STUDENT_ID_LAST4 | 2211 | 本人学号后4位，用于计算StudentID校验字段 |
| WINDOW_SIZE | 400 | 发送窗口大小，单位字节，固定400字节符合任务要求 |
| PACKET_SIZE | 80 | 单个UDP数据包的数据部分长度，单位字节 |
| TOTAL_PACKETS | 30 | 待发送的数据包总数 |
| TIMEOUT | 0.3 | 超时重传时间，单位秒，默认300ms |
| LOG_FILE | run_log.txt | 日志文件名 |

 服务器端（udpserver.py）
| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| SERVER_PORT | 10000 | 服务器监听的UDP端口 |
| PACKET_LOSS_RATE | 0.2 | 模拟丢包率，默认20% |
| STUDENT_ID_XOR | 0x5A3C | StudentID校验异或固定值，与任务要求一致 |
| LOG_FILE | run_log.txt | 日志文件名 |

 四、运行步骤
 1. 服务器端（虚拟机内操作）
(1) 将`udpserver.py`文件复制到虚拟机中
(2) 打开命令提示符，进入文件所在目录
(3) 执行命令启动服务器：

python udpserver.py

(4) 启动成功后控制台显示：`服务器已启动，监听端口 10000`，程序保持后台运行，等待客户端连接

 2. 客户端（主机内操作）
(1) 提前验证主机与虚拟机网络连通：打开主机命令提示符，执行`ping 虚拟机IP地址`，能收到回复则网络正常
(2) 进入`udpclient.py`所在目录
(3) 执行命令启动客户端，格式为：

python udpclient.py <服务器IP地址> <服务器端口>

- 本地测试示例：

python udpclient.py 127.0.0.1 10000

- 虚拟机测试示例（替换为实际虚拟机IP）：

python udpclient.py 192.168.26.130 10000

(4) 程序自动完成「连接建立 → 数据传输 → 超时重传 → 断开连接」全流程，传输完成后控制台输出丢包率、RTT统计等汇总信息

 3. Wireshark抓包验证（可选，用于验收）
(1) 主机打开Wireshark，选择对应网卡：桥接模式选物理上网网卡，NAT模式选VMware Network Adapter VMnet8
(2) 在过滤栏输入过滤条件：`udp.port == 10000`，按下回车
(3) 点击开始抓包按钮，再运行客户端程序
(4) 传输完成后停止抓包，可对照`run_log.txt`验证报文时间戳、报文类型、数据长度的一致性

五、自定义应用层协议设计
 报文首部格式（固定8字节）
| 字段名     | 长度（字节） |    说明   |
|------------|-------------|---------|
| 报文类型   | 1 | 标识报文功能：0x01=连接请求，0x02=连接响应，0x03=数据报文，0x04=确认报文，0x05=断开连接 |
| StudentID  | 2 | 无符号短整型，值为学号后4位与0x5A3C异或运算结果，用于服务器身份校验 |
| 序列号     | 2 | 数据报文的起始字节编号，标识本报文数据在整体数据流中的位置 |
| 确认号     | 2 | 累积确认号，表示该编号之前的所有字节均已正确接收 |
| 数据长度   | 1 | 报文携带的数据部分长度，单位字节，控制报文该字段为0 |

 交互流程
1. 连接建立：客户端发送携带StudentID的连接请求 → 服务器校验身份，合法则返回连接响应，连接建立完成
2. 数据传输：客户端按窗口大小批量发送数据包 → 服务器按设定概率随机丢包 → 按序到达的数据更新预期序列号，返回累积确认 → 客户端收到确认后滑动窗口，继续发送新数据
3. 超时重传：若客户端在超时时间内未收到确认，判定为丢包，重传窗口内所有未确认的数据包
4. 断开连接：所有数据传输完成后，客户端发送断开连接请求 → 服务器响应后释放连接资源

 六、功能特性
1. 命令行参数传参：客户端启动时通过命令行指定服务器IP和端口，代码无硬编码地址
2. StudentID身份校验：严格按照任务规则实现异或校验，非法客户端无法建立连接
3. GBN滑动窗口：固定400字节发送窗口，实现回退N帧协议的流水线传输
4. 应用层丢包模拟：服务器端按设定概率随机丢弃数据包，模拟不可靠网络环境
5. 超时重传机制：客户端300ms超时后自动重传窗口内所有未确认数据
6. 累积确认机制：服务器采用累积确认，一次性确认所有按序到达的数据
7. RTT统计分析：自动计算每个数据包的往返时间，最终输出最大值、最小值、平均值、标准差
8. 自动日志记录：生成带毫秒级时间戳的运行日志，所有发包、收包、超时、重传事件可追溯
9. 服务器时间回显：确认报文中携带服务器系统时间，客户端接收后打印输出

 七、注意事项
1. 运行顺序：必须先启动服务器端，再运行客户端，否则会出现连接超时
2. 防火墙设置：虚拟机需开放对应UDP端口（默认10000），或临时关闭防火墙，避免数据包被拦截
3. 端口一致性：若修改服务器监听端口，客户端运行时必须传入相同的端口参数
4. 日志覆盖：每次运行客户端会清空并重建`run_log.txt`，如需保存历史日志请提前备份
5. 网络模式：虚拟机推荐使用桥接模式，确保主机与虚拟机在同一网段；使用NAT模式需配置端口映射
6. 丢包率说明：最终实际丢包率受随机数影响，与设定的20%存在合理波动，属于正常现象
